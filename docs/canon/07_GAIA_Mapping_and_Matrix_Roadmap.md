## 1. What Is a GAIA Matrix?

A matrix in GAIA is a structured canonical document that maps a specific domain across multiple dimensions simultaneously. Unlike a specification (which defines how a system works) or a doctrine (which defines what GAIA values), a matrix answers the question:

> **How does this domain look when all its relevant dimensions are laid out together and cross-referenced?**

A well-formed GAIA matrix has:
- A clearly defined domain
- A set of dimensions (axes of variation)
- A mapping of entities or states across those dimensions
- Cross-references to the canonical documents that govern each cell
- A descent stack anchor

## 3. Matrix Governance Rules

1. A matrix is canonical when committed to `docs/canon/` with status CANONICAL in C00.
2. Matrices must cross-reference the descent stack layer they anchor to.
3. Where a matrix cell refers to an entity defined in another canonical document, the cross-reference must be explicit.
4. Matrices may be extended by amendment with a version increment and CHANGELOG entry.
5. A matrix may not contradict a specification or doctrine. Where conflict exists, the specification or doctrine governs.
