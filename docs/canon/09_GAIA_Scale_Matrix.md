## 2. Scale Crossing Rules

### SC-1: Declare the Scale
Every GAIA action must declare its primary operating scale before execution.

### SC-2: No Upward Aggregation Without Consent
Data at scale S-n may not be aggregated to scale S-(n+k) without explicit consent at each intermediate scale.

### SC-3: No Downward Imposition Without Subsidiarity Check
Decisions at scale S-n may not be imposed on S-(n-k) if those entities can govern the matter themselves.

### SC-4: Scale Mismatch Is a Design Error
Applying S7 governance logic to an S3 context, or S3 consent mechanisms to an S9 decision, is a design violation.

### SC-5: Planetary Scale Requires Constitutional Authority
Any action with S9 or S10 consequences requires D-series constitutional authority and Codex Decision Test ratification.
